# Report template field sources

BUILI's `buili.project-record.v3` family is an original layout. Its operational fields were normalized from current primary product documentation rather than copied from a marketplace design.

- Autodesk Build RFI fields and RFI reports: <https://help.autodesk.com/cloudhelp/ENU/Build-Rfis/files/admin-rfis/RFI_Types.html> and <https://help.autodesk.com/cloudhelp/ENU/Docs-Reports/files/report-types/RFI_Reports.html>
- Autodesk Build issue detail reports: <https://help.autodesk.com/view/BUILD/ENU/?guid=Issue_Reports>
- Procore punch items: <https://support.procore.com/products/online/user-guide/project-level/punch-list/tutorials/create-a-punch-list-item>
- Procore change events: <https://support.procore.com/products/online/user-guide/project-level/change-events/tutorials/create-change-events>
- Procore daily log: <https://support.procore.com/products/online/user-guide/project-level/daily-log/tutorials/daily-log-overview>

Each BUILI document kind has its own required-field schema and narrative section set. Source revision, field evidence, review state, immutable artifact hash, and activity history remain shared traceability controls.
